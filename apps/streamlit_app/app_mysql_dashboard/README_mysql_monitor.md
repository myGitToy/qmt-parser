# MySQL数据库监控WebUI使用说明

## 功能特点

✅ **实时监控**: 监控MySQL数据库连接状态、性能指标和进程
✅ **多数据库支持**: 自动从.env文件读取多个数据库配置
✅ **可视化界面**: 使用图表和仪表盘展示监控数据
✅ **自动刷新**: 支持定时自动刷新监控数据
✅ **响应式设计**: 适配不同屏幕尺寸

## 监控指标

### 📊 概览页面
- 当前连接数
- 连接使用率
- 历史最大连接数
- 连接使用率仪表盘

### 🔗 连接状态
- 详细连接统计
- 连接数趋势图
- 连接状态历史

### ⚡ 性能指标
- InnoDB缓冲池状态
- 缓冲池命中率
- 行操作统计(增删改查)
- 内存使用情况

### 📁 存储信息
- 各数据库大小分布
- 表大小排行
- 存储空间使用情况

### 🔍 进程监控
- 当前活跃进程
- 长时间运行查询
- 进程状态分布
- 详细进程列表

## 安装和使用

### 1. 安装依赖
```bash
pip install -r requirements_monitor.txt
```

### 2. 配置数据库连接
确保项目根目录下的`.env`文件包含数据库连接信息：
```
UBUNTU186_DB_CONN="mysql+pymysql://root:password@192.168.1.186:3306/stock"
LOCAL_DB_CONN="mysql+pymysql://root:password@localhost:3306/stock"
# ... 其他数据库连接
```

### 3. 启动应用

#### 方法一：使用批处理文件（Windows）
```bash
start_mysql_monitor.bat
```

#### 方法二：手动启动
```bash
streamlit run mysql_monitor.py --server.port 8501
```

### 4. 访问监控界面
打开浏览器访问：http://localhost:8501

## 界面操作

### 侧边栏
- **数据库选择**: 从.env文件中加载的数据库列表中选择
- **连接信息**: 显示当前选择数据库的连接详情
- **监控设置**: 配置自动刷新间隔
- **连接按钮**: 连接到选择的数据库

### 主界面标签页
1. **📊 概览**: 关键指标概览和连接使用率仪表盘
2. **🔗 连接状态**: 详细连接统计和趋势分析
3. **⚡ 性能指标**: InnoDB性能指标和缓冲池状态
4. **📁 存储信息**: 数据库和表的存储使用情况
5. **🔍 进程监控**: 实时进程监控和查询分析

## 注意事项

### 数据库权限
监控用户需要以下权限：
```sql
GRANT SELECT, PROCESS, SHOW DATABASES ON *.* TO 'monitor_user'@'%';
GRANT SELECT ON performance_schema.* TO 'monitor_user'@'%';
GRANT SELECT ON information_schema.* TO 'monitor_user'@'%';
```

### 安全建议
- 建议为监控创建专门的只读用户
- 不要在生产环境使用root用户进行监控
- 确保.env文件不被提交到版本控制系统

### 性能影响
- 监控查询设计为轻量级，对数据库性能影响最小
- 建议监控刷新间隔设置为10秒以上
- 在高负载时可以关闭自动刷新

## 故障排除

### 连接失败
1. 检查.env文件中的数据库连接字符串格式
2. 确认数据库服务器地址和端口可访问
3. 验证用户名和密码正确
4. 检查数据库用户权限

### 数据显示异常
1. 确认用户有足够的查询权限
2. 检查MySQL版本兼容性
3. 查看Streamlit控制台错误信息

### 性能问题
1. 减少自动刷新频率
2. 检查网络延迟
3. 优化数据库服务器性能
